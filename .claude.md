# Project Skills and Guidelines

## React/Next.js Best Practices

When working with React or Next.js code in this project, apply Vercel's performance optimization guidelines:

### Critical Priorities

1. **Eliminate Waterfalls**
   - Use Promise.all() for independent async operations
   - Move await into branches where actually used
   - Start promises early, await late

2. **Bundle Size Optimization**
   - Import directly, avoid barrel files (e.g., `import { Button } from '@/components/Button'` not `from '@/components'`)
   - Use next/dynamic for heavy components
   - Load analytics/logging after hydration

3. **Server-Side Performance**
   - Use React.cache() for per-request deduplication
   - Minimize data passed to client components
   - Use after() for non-blocking operations

4. **Re-render Optimization**
   - Extract expensive work into memoized components
   - Use primitive dependencies in effects
   - Use startTransition for non-urgent updates
