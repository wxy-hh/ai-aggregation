# 🚀 音频存储功能快速开始

## 功能概述

现在音频历史记录支持智能存储：

- **最近10条**：保存完整音频 + 文本
- **超过10条**：只保存文本（转录 + 翻译）

## 立即体验

### 1. 启动应用

```bash
pnpm dev
```

访问：http://localhost:3000/voice

### 2. 上传音频

1. 点击"上传音频"标签
2. 选择或拖拽音频文件（MP3/WAV/AAC）
3. 等待转录和翻译完成

### 3. 查看历史记录

1. 右侧面板显示所有历史记录
2. 点击任意记录查看详情
3. **最近10条**：可以播放音频 🎵
4. **超过10条**：显示"音频不可用"，但文本完整 📝

### 4. 测试自动清理

上传 12 个音频文件，然后检查：

- 第 1-10 条：有音频播放器 ✅
- 第 11-12 条：显示"音频不可用" ⚠️

## 开发者工具检查

### 查看 IndexedDB

1. 打开浏览器开发者工具（F12）
2. 切换到 **Application** 标签
3. 展开 **IndexedDB** → **AudioHistoryDB** → **audioHistory**
4. 查看记录：
   - 最近10条有 `audioBlob` 字段（Blob 对象）
   - 超过10条的 `audioBlob` 为空

### 查看存储使用

在控制台运行：

```javascript
navigator.storage.estimate().then((estimate) => {
  console.log('已使用:', (estimate.usage / 1024 / 1024).toFixed(2), 'MB');
  console.log('总配额:', (estimate.quota / 1024 / 1024).toFixed(2), 'MB');
});
```

预期结果：约 50MB（10个音频文件）

## 配置选项

### 修改保留数量

编辑 `apps/web/src/lib/storage/indexeddb-storage.ts`：

```typescript
// 当前默认：保留最近 10 条
const KEEP_AUDIO_COUNT = 10;

// 可以改为其他值
const KEEP_AUDIO_COUNT = 20; // 保留最近 20 条
```

### 完全禁用音频存储

如果不想保存任何音频：

```typescript
const KEEP_AUDIO_COUNT = 0; // 不保存任何音频
```

## 数据迁移（如果有旧数据）

如果你已经有历史记录，运行迁移清理旧音频：

```javascript
// 在浏览器控制台运行
await audioStorageMigration.checkStatus(); // 查看状态
await audioStorageMigration.runMigration(); // 运行迁移
```

## 故障排除

### 问题：所有记录都没有音频

**原因**：可能是旧数据，需要重新上传

**解决**：

1. 上传新的音频文件
2. 新上传的会自动保存音频

### 问题：存储空间不足

**原因**：浏览器存储配额已满

**解决**：

1. 清除浏览器数据
2. 或删除一些历史记录
3. 或减少 `KEEP_AUDIO_COUNT` 值

### 问题：音频播放失败

**原因**：Blob URL 创建失败

**解决**：

1. 刷新页面重试
2. 检查浏览器控制台错误
3. 确认音频格式支持
