## 核心模块

### Web 应用 (apps/web)

基于 Next.js 15 的全栈应用：

- **对话界面**: 支持流式输出的 AI 对话
- **任务中心**: 查看和管理异步任务
- **文件管理**: 上传音频、下载生成的文件
- **用户系统**: 登录、注册、用量统计

技术选型：
- Next.js 15 (App Router)
- React 19
- Tailwind CSS
- React Query (数据获取与缓存)

### Worker 服务 (apps/worker)

异步任务处理服务：

- **STT Worker**: 语音转文字
- **PPT Worker**: PPT 生成
- **Image Worker**: 图像生成

特点：
- 基于 BullMQ 的可靠任务队列
- 支持重试与错误处理
- 并发控制与资源管理

### 共享包 (packages/*)

#### shared
- 类型定义
- 错误码
- Zod schemas

#### providers
- AI 提供方统一接口
- 支持通义千问、智谱、DeepSeek
- 自动故障转移

#### db
- Prisma schema
- 数据库迁移
- Client 封装

#### queue
- 队列定义
- Job 类型
- 重试策略

#### storage
- 对象存储抽象
- 支持 OSS/COS/MinIO

#### logger
- 结构化日志
- Trace ID 支持

## 数据流

### 对话流程

```
用户输入 → Web API → Provider 适配 → AI 模型
                                        ↓
用户界面 ← SSE 流式输出 ← Provider 适配 ← AI 响应
```

### 异步任务流程

```
用户请求 → Web API → 创建任务 → 入队
                        ↓
                   返回任务 ID
                        ↓
Worker 消费 → 调用外部服务 → 更新任务状态
                                ↓
用户轮询/WebSocket ← 任务完成通知
```

## 扩展性设计

### 水平扩展

- Web: 无状态设计，可多实例部署
- Worker: 通过增加实例提升并发能力
- 数据库: 读写分离、连接池

### 模型扩展

通过 Provider 接口添加新模型：

```typescript
class NewProvider implements AIProvider {
  async chat(options) { /* ... */ }
  async *chatStream(options) { /* ... */ }
}
```

### 任务类型扩展

1. 在 `packages/queue` 定义新 job 类型
2. 在 `apps/worker` 实现新 worker
3. 在 `apps/web` 添加创建任务的 API

## 安全设计

- **鉴权**: JWT / Session
- **限流**: Redis + 滑动窗口
- **配额**: 用户级别的用量控制
- **输入验证**: Zod schema 验证
- **SQL 注入防护**: Prisma ORM
- **XSS 防护**: React 自动转义

## 监控与可观测

- **日志**: 结构化 JSON 日志
- **错误追踪**: Sentry
- **性能监控**: Next.js Analytics
- **队列监控**: BullMQ Dashboard
- **数据库监控**: Prisma Metrics

## 成本优化

- **模型选择**: 主备模型策略，优先使用性价比高的模型
- **缓存**: Redis 缓存常见对话
- **限流**: 防止滥用
- **资源池**: Worker 并发控制
- **对象存储**: 生命周期管理，自动清理过期文件
