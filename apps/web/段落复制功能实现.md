# 段落复制功能实现

## 🎯 功能说明

为每个段落添加复制按钮，支持：

1. **鼠标悬停显示**：鼠标移到段落上时显示复制按钮
2. **一键复制**：点击按钮复制当前段落文本
3. **视觉反馈**：复制成功后显示对勾图标和绿色背景
4. **自动恢复**：2秒后自动恢复到初始状态

## 🔧 实现原理

### 1. 添加状态管理

```typescript
function SegmentBlock({ segment, showTranslation, ... }) {
  const [copied, setCopied] = useState(false);

  // ...
}
```

**状态说明**：

- `copied = false`：未复制状态，显示复制图标
- `copied = true`：已复制状态，显示对勾图标

### 2. 实现复制逻辑

```typescript
const handleCopy = async (e: React.MouseEvent) => {
  e.stopPropagation(); // 阻止触发段落点击

  // 根据显示模式选择复制内容
  const textToCopy = showTranslation
    ? segment.translatedText // 译文
    : segment.originalText; // 原文

  try {
    // 使用 Clipboard API 复制
    await navigator.clipboard.writeText(textToCopy);

    // 设置复制成功状态
    setCopied(true);

    // 2秒后恢复
    setTimeout(() => {
      setCopied(false);
    }, 2000);
  } catch (error) {
    console.error('复制失败:', error);
  }
};
```

**关键点**：

1. **阻止事件冒泡**：`e.stopPropagation()` 防止触发段落点击
2. **智能选择内容**：根据 `showTranslation` 决定复制原文还是译文
3. **异步复制**：使用 `async/await` 处理 Clipboard API
4. **自动恢复**：2秒后自动恢复到初始状态

### 3. 渲染复制按钮

```typescript
<div className="absolute top-2 right-2 opacity-0 group-hover:opacity-100 transition-opacity">
  <button
    onClick={handleCopy}
    className={cn(
      'w-8 h-8 rounded-md shadow-sm flex items-center justify-center transition-all',
      copied
        ? 'bg-green-500 text-white'  // 复制成功：绿色背景
        : 'bg-white hover:bg-blue-50' // 初始状态：白色背景
    )}
    title={copied ? '已复制' : '复制'}
  >
    {copied ? (
      // 对勾图标
      <svg>...</svg>
    ) : (
      // 复制图标
      <svg>...</svg>
    )}
  </button>
</div>
```

**样式说明**：

- `opacity-0 group-hover:opacity-100`：鼠标悬停时显示
- `absolute top-2 right-2`：定位在右上角
- `transition-all`：平滑过渡动画

## 🔍 问题排查

### 如果复制按钮不显示

**检查点1：查看 CSS 类**

```typescript
className = 'opacity-0 group-hover:opacity-100';
```

确认：

- 父元素有 `group` 类
- 按钮有 `opacity-0` 和 `group-hover:opacity-100`

**检查点2：查看浏览器开发者工具**

1. 打开 Elements 标签
2. 选中段落元素
3. 查看复制按钮的样式
4. 确认 `opacity` 值

### 如果复制失败

**检查点1：查看控制台错误**

```javascript
console.error('复制失败:', error);
```

**可能原因**：

1. **浏览器不支持 Clipboard API**
   - 解决：使用 HTTPS 或 localhost
   - Clipboard API 只在安全上下文中可用

2. **权限被拒绝**
   - 解决：检查浏览器权限设置

3. **文本为空**
   - 解决：检查 `segment.originalText` 或 `segment.translatedText`

**检查点2：测试 Clipboard API**

在浏览器控制台输入：

```javascript
navigator.clipboard
  .writeText('测试')
  .then(() => {
    console.log('复制成功');
  })
  .catch((err) => {
    console.error('复制失败:', err);
  });
```

### 如果复制了错误的内容

**检查点1：查看 showTranslation 值**

```typescript
const textToCopy = showTranslation ? segment.translatedText : segment.originalText;

console.log('showTranslation:', showTranslation);
console.log('复制内容:', textToCopy);
```

**检查点2：查看段落数据**

```typescript
console.log('segment:', segment);
console.log('originalText:', segment.originalText);
console.log('translatedText:', segment.translatedText);
```

### 如果按钮不恢复

**检查点1：查看 setTimeout**

```typescript
setTimeout(() => {
  console.log('恢复初始状态');
  setCopied(false);
}, 2000);
```

**检查点2：查看组件是否卸载**

如果组件在2秒内卸载，setTimeout 可能不会执行。

**解决方案**：使用 useEffect 清理

```typescript
useEffect(() => {
  let timeoutId: NodeJS.Timeout;

  if (copied) {
    timeoutId = setTimeout(() => {
      setCopied(false);
    }, 2000);
  }

  return () => {
    if (timeoutId) {
      clearTimeout(timeoutId);
    }
  };
}, [copied]);
```
