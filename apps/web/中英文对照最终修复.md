# 中英文对照最终修复

## 🐛 发现的新问题

### 问题 1：中英文仍然不对照

**原因分析**：

虽然改为了逐句翻译，但是在 `createSegments` 函数中仍然使用了错误的分割逻辑：

```typescript
// ❌ 错误的做法
const translatedSentences = translationText.split(/[.!?]/);

// 问题：
// 1. 逐句翻译后，结果是用 '. ' 连接的字符串
// 2. 再次按 '.' 分割会导致错乱
// 3. 例如："Hello. This is good." 分割后变成 ["Hello", " This is good", ""]
```

**示例**：

```
逐句翻译结果：
translatedSentences = [
  "Hello, welcome to our product",
  "This is the first feature",
  "Next is the second feature"
]

合并后：
translationText = "Hello, welcome to our product. This is the first feature. Next is the second feature."

再次分割：
split(/[.!?]/) = [
  "Hello, welcome to our product",
  " This is the first feature",    // ⚠️ 前面多了空格
  " Next is the second feature",   // ⚠️ 前面多了空格
  ""                                // ⚠️ 多了空字符串
]

✅ 虽然数量对上了，但是有空格和空字符串问题
```

### 问题 2：点击选中错误的 div

**原因分析**：

在双栏布局中，每行有两个 `SegmentBlock`（原文和译文），但是它们共享同一个 `onClick` 事件处理：

```typescript
// ❌ 问题代码
<div className="grid grid-cols-2 gap-6">
  <SegmentBlock onClick={() => onSeek(segment.startTime)} />  // 左栏
  <SegmentBlock onClick={() => onSeek(segment.startTime)} />  // 右栏
</div>

// 问题：
// 1. 点击左栏，触发左栏的 onClick
// 2. 但是由于事件冒泡，可能触发父元素的事件
// 3. 导致选中了错误的元素
```

**用户反馈**：

> "点击当前中文或者英文那列，选中的是目标div上面的那列"

这说明点击事件被错误地传播或处理了。

## ✅ 最终修复方案

### 修复 1：直接传递翻译数组

**核心改进**：不再合并和分割翻译结果，直接传递数组

```typescript
// ✅ 新的做法

// 1. 修改 createSegments 函数签名
const createSegments = (
  transcriptionText: string,
  translationTexts?: string[], // 改为数组
  audioDuration?: number
) => {
  // ...

  // 2. 直接使用数组索引获取翻译
  const translatedText = translationTexts?.[index] || 'Translation in progress...';

  return {
    originalText: sentence.trim() + '。',
    translatedText, // 直接使用，不需要分割
    // ...
  };
}

// 3. 修改翻译逻辑，保存数组
const translatedSentences: string[] = [];
for (let i = 0; i < sentences.length; i++) {
  const result = await translateText({ text: sentences[i], ... });
  translatedSentences.push(result.translatedText.trim());
}

// 4. 直接保存数组，不合并
setTranslationResults(translatedSentences); // 保存数组

// 5. 调用时传入数组
const segments = createSegments(
  transcriptionResult,
  translationResults, // 传入数组
  audioDuration
);
```

**优点**：

- ✅ 完全避免了分割问题
- ✅ 保证一一对应
- ✅ 代码更清晰

### 修复 2：添加 onClick 参数和事件阻止

**核心改进**：为每个 `SegmentBlock` 添加独立的 `onClick` 处理

```typescript
// ✅ 新的做法

// 1. 修改 SegmentBlock 组件，添加 onClick 参数
function SegmentBlock({
  segment,
  isActive,
  showTranslation,
  onSeek,
  onClick, // 新增
}: {
  segment: TranscriptSegment;
  isActive: boolean;
  showTranslation: boolean;
  onSeek: (time: number) => void;
  onClick?: (e: React.MouseEvent) => void; // 新增
}) {
  return (
    <div
      onClick={onClick || (() => onSeek(segment.startTime))}
      // ...
    >
      {/* ... */}
    </div>
  );
}

// 2. 在渲染时传入独立的 onClick
{segments.map((segment) => (
  <div className="grid grid-cols-2 gap-6">
    {/* 左栏 - 原文 */}
    <SegmentBlock
      segment={segment}
      onClick={(e) => {
        e.stopPropagation(); // 阻止事件冒泡
        handleSeek(segment.startTime);
      }}
    />

    {/* 右栏 - 译文 */}
    <SegmentBlock
      segment={segment}
      onClick={(e) => {
        e.stopPropagation(); // 阻止事件冒泡
        handleSeek(segment.startTime);
      }}
    />
  </div>
))}
```

**优点**：

- ✅ 每个块有独立的点击处理
- ✅ `e.stopPropagation()` 阻止事件冒泡
- ✅ 点击哪个就选中哪个
