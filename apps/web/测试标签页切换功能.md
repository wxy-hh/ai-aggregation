# 测试标签页切换功能

## 🎯 测试目标

验证在上传音频处理过程中切换标签页时：

1. ✅ 可以正常切换
2. ✅ 处理任务继续执行
3. ✅ 切换回来显示最新状态
4. ✅ 不会回到初始上传界面

## 🚀 快速测试

### 测试 1：处理中切换

1. 访问：http://localhost:3002/voice
2. 点击"上传音频"标签
3. 上传测试音频：`apps/web/src/assets/voice/人保律师-2510291614.mp3`
4. **立即切换到"实时录音"标签**（不等处理完成）
5. **观察"上传音频"标签**：
   - ✅ 应该看到蓝色脉冲圆点（表示正在处理）
6. 等待 1-2 分钟
7. **观察"上传音频"标签**：
   - ✅ 蓝色脉冲变为绿色圆点（表示处理完成）
8. 切换回"上传音频"标签
9. **观察界面**：
   - ✅ 直接显示处理结果
   - ✅ 不是初始的上传界面
   - ✅ 可以看到转录和翻译结果

### 测试 2：多次切换

1. 在"上传音频"标签上传文件
2. 切换到"实时录音"
3. 切换回"上传音频"
4. 再次切换到"实时录音"
5. 再次切换回"上传音频"
6. **观察**：
   - ✅ 每次切换回来都显示相同的状态
   - ✅ 不会重置到初始界面

### 测试 3：未上传时切换

1. 刷新页面
2. 点击"上传音频"标签（不上传文件）
3. 切换到"实时录音"
4. **观察"上传音频"标签**：
   - ✅ 无指示器（正常）
5. 切换回"上传音频"
6. **观察**：
   - ✅ 显示初始的上传界面（正常）

## ❌ 失败情况

### 如果切换回来显示初始界面

**现象**：

- 切换回"上传音频"标签
- 看到初始的上传界面（拖拽区域）
- 之前的结果消失

**原因**：

- 组件被卸载了
- 使用了条件渲染而不是 CSS 隐藏

**检查**：

```typescript
// 错误的方式
{mode === 'upload' && <UploadAudio />}

// 正确的方式
<div className={cn(mode === 'upload' ? 'block' : 'hidden')}>
  <UploadAudio />
</div>
```

### 如果没有状态指示器

**现象**：

- 切换到"实时录音"后
- "上传音频"标签没有指示器

**原因**：

- 状态回调没有正确设置
- 父组件没有接收到状态更新

**检查控制台**：

```
应该看到：
📊 上传状态变化: { ... }
📊 父组件收到状态更新: { ... }

如果没有，检查 onStateChange 回调
```
