# 音频转文本完整流程说明

## 📋 当前实现流程

### 完整流程图

```
用户上传音频文件 (浏览器)
    │
    ├─> 1. 前端验证
    │   ├─ 文件类型检查 (MP3/WAV/AAC)
    │   ├─ 文件大小检查 (<50MB)
    │   └─ 创建 FormData
    │
    ├─> 2. 发送到后端 API
    │   POST /api/voice/transcribe
    │   Content-Type: multipart/form-data
    │
    ├─> 3. 后端接收文件
    │   ├─ 解析 FormData
    │   ├─ 再次验证文件
    │   └─ 保存到临时目录
    │       └─ /tmp/voice-uploads/1234567890-audio.mp3
    │
    ├─> 4. 调用 SiliconFlow API
    │   ├─ 读取临时文件
    │   ├─ 创建 multipart/form-data
    │   ├─ 发送到 SiliconFlow
    │   │   POST https://api.siliconflow.cn/v1/audio/transcriptions
    │   │   Authorization: Bearer <API_KEY>
    │   │   Body: file + model
    │   └─ 接收响应
    │       └─ { "text": "转录的中文文本" }
    │
    ├─> 5. 清理临时文件
    │   └─ 删除 /tmp/voice-uploads/1234567890-audio.mp3
    │
    ├─> 6. 返回给前端
    │   └─ { "transcription": "转录的中文文本" }
    │
    ├─> 7. 前端显示转录结果
    │   └─ 显示在界面上
    │
    └─> 8. 自动调用翻译 API
        ├─ POST /api/voice/translate
        ├─ Body: { "text": "转录的中文文本" }
        ├─ 调用讯飞Lite模型
        └─ 返回英文翻译
```

### 步骤3：保存到临时目录

**文件位置**：`apps/web/src/lib/file-upload.ts`

```typescript
export async function saveUploadedFile(file: File): Promise<string> {
  // 3.1 创建临时目录
  const TEMP_DIR = '/tmp/voice-uploads';
  if (!existsSync(TEMP_DIR)) {
    await fs.mkdir(TEMP_DIR, { recursive: true });
  }

  // 3.2 生成唯一文件名
  const uniqueFileName = `${Date.now()}-${file.name}`;
  const filePath = path.join(TEMP_DIR, uniqueFileName);
  // 例如: /tmp/voice-uploads/1738483200000-audio.mp3

  // 3.3 保存文件到磁盘
  const buffer = Buffer.from(await file.arrayBuffer());
  await fs.writeFile(filePath, buffer);

  return filePath;
}
```

**关键点**：

- ✅ **这是唯一保存到磁盘的地方**
- ✅ 保存到临时目录 `/tmp/voice-uploads/`
- ✅ 文件名格式：`时间戳-原文件名.mp3`
- ⚠️ 这是**临时文件**，处理完会删除

**为什么要保存到临时文件？**

- SiliconFlow API 需要读取文件内容
- 使用 `readFileSync()` 读取文件
- 无法直接传递内存中的 File 对象

### 步骤4：调用 SiliconFlow API

**文件位置**：`apps/web/src/lib/siliconflow.ts`

```typescript
export async function transcribeAudio(
  filePath: string, // 临时文件路径
  model: string
): Promise<TranscribeResult> {
  // 4.1 读取临时文件
  const fileContent = readFileSync(filePath);
  const fileName = basename(filePath);

  // 4.2 创建 multipart/form-data
  const boundary = '----Boundary' + Date.now();
  const parts: Buffer[] = [];

  // 添加 file 字段
  parts.push(
    Buffer.from(
      `--${boundary}\r\n` +
        `Content-Disposition: form-data; name="file"; filename="${fileName}"\r\n` +
        `Content-Type: audio/mpeg\r\n\r\n`,
      'utf8'
    )
  );
  parts.push(fileContent); // 文件内容
  parts.push(Buffer.from('\r\n', 'utf8'));

  // 添加 model 字段
  parts.push(
    Buffer.from(
      `--${boundary}\r\n` +
        `Content-Disposition: form-data; name="model"\r\n\r\n` +
        `${model}\r\n` +
        `--${boundary}--\r\n`,
      'utf8'
    )
  );

  const body = Buffer.concat(parts);

  // 4.3 发送到 SiliconFlow
  const response = await fetch('https://api.siliconflow.cn/v1/audio/transcriptions', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${API_KEY}`,
      'Content-Type': `multipart/form-data; boundary=${boundary}`,
    },
    body: body,
  });

  // 4.4 解析响应
  const result = await response.json();
  // { "text": "你好，欢迎使用语音转录系统。这是一个测试。" }

  return result;
}
```

**关键点**：

- ✅ 从临时文件读取内容
- ✅ **直接调用 SiliconFlow API**
- ✅ **不生成中间文本文件**
- ✅ 只读取 API 返回的 JSON 结果

### 步骤5：清理临时文件

**文件位置**：`apps/web/src/app/api/voice/transcribe/route.ts`

```typescript
try {
  // 调用 SiliconFlow API
  const result = await transcribeAudio(tempFilePath, model);

  // 5.1 删除临时文件
  await deleteFile(tempFilePath);
  tempFilePath = null;

  // 5.2 返回结果
  return NextResponse.json({
    transcription: result.text,
  });
} catch (error) {
  // 5.3 出错也要删除临时文件
  if (tempFilePath) {
    await deleteFile(tempFilePath);
  }
  throw error;
}
```

**关键点**：

- ✅ 处理完立即删除临时文件
- ✅ 即使出错也会删除
- ✅ 不留下垃圾文件

## ✅ 回答你的问题

### 问题：先转成文本文件，再转成文字吗？还是直接调用模型接口？

**答案：直接调用模型接口，只读取返回的结果** ✅

**详细说明**：

1. **不生成中间文本文件**
   - ❌ 不会生成 `.txt` 文件
   - ❌ 不会保存转录结果到文件
   - ✅ 只在内存中处理

2. **唯一的文件操作**
   - ✅ 只保存**临时音频文件**到 `/tmp/voice-uploads/`
   - ✅ 这是为了调用 SiliconFlow API（需要读取文件）
   - ✅ 处理完立即删除

3. **数据流向**

   ```
   音频文件 (浏览器)
     → 临时音频文件 (服务器磁盘) ⚠️ 唯一的文件
     → SiliconFlow API
     → JSON 响应 (内存)
     → 前端显示 (内存)
   ```

4. **没有中间文本文件**
   ```
   ❌ 不会生成: /tmp/transcription-result.txt
   ❌ 不会生成: /tmp/translation-result.txt
   ✅ 只有: /tmp/voice-uploads/1234567890-audio.mp3 (临时)
   ```

## 🔍 为什么要保存临时音频文件？

**原因**：

- SiliconFlow API 需要 `multipart/form-data` 格式
- 需要读取文件内容创建 Buffer
- 使用 Node.js 的 `readFileSync()` 读取

**不能直接传递吗？**

- ❌ 不能直接传递浏览器的 File 对象
- ❌ File 对象在服务器端需要转换
- ✅ 保存到临时文件是最简单的方式

**为什么不保留文件？**

- 节省磁盘空间
- 避免隐私泄露
- 符合临时文件的设计原则

## 📝 总结

### 当前流程特点

✅ **优点**：

- 简单直接
- 不生成多余文件
- 自动清理临时文件
- 内存处理效率高

⚠️ **注意事项**：

- 临时文件在 `/tmp/voice-uploads/`
- 处理完自动删除
- 如果服务器崩溃，可能留下临时文件
- 可以定期清理旧文件（已实现 `cleanupOldFiles` 函数）

### 完整流程总结

```
用户上传音频
  ↓
保存到临时文件 (/tmp/voice-uploads/)
  ↓
调用 SiliconFlow API (直接读取临时文件)

  ↓

  ↓
显示在界面上
  ↓
自动调用翻译 API (文本 → 文本)
  ↓
显示双语结果
```

**关键点**：

- ✅ 只有一个临时音频文件
- ✅ 没有中间文本文件
- ✅ 直接调用 API，读取 JSON 响应
- ✅ 所有文本处理都在内存中
