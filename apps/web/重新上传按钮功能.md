# 重新上传按钮功能

## 🎯 功能说明

在转录结果页面添加"重新上传"按钮，允许用户：

1. **快速返回上传界面**：无需刷新页面
2. **清除当前结果**：自动清理转录和翻译数据
3. **重新选择文件**：可以上传新的音频文件

## 🎨 设计原则

遵循 UI/UX 最佳实践：

### 1. 位置合理

- **放置在 Header 的 Actions 区域**
- **位于最左侧**：作为主要操作之一
- **与其他按钮对齐**：保持视觉一致性

### 2. 样式美观

- **Outline 风格**：区别于主要操作按钮
- **上传图标**：直观表达功能
- **悬停效果**：提供视觉反馈
- **响应式设计**：适配不同屏幕尺寸

### 3. 交互友好

- **清晰的文字标签**："重新上传"
- **图标 + 文字**：提高可识别性
- **平滑过渡动画**：提升用户体验

## 🔧 实现细节

### 1. 组件 Props

```typescript
interface TranscriptionResultProps {
  fileName: string;
  language: string;
  targetLanguage: string;
  segments: TranscriptSegment[];
  audioUrl?: string;
  isTranslating?: boolean;
  onReupload?: () => void; // 新增：重新上传回调
}
```

### 2. 按钮实现

```typescript
{/* 重新上传按钮 */}
{onReupload && (
  <Button
    variant="outline"
    size="sm"
    onClick={onReupload}
    className="gap-2 border-slate-200 dark:border-slate-700 text-slate-700 dark:text-slate-300 hover:bg-slate-50 dark:hover:bg-slate-800 hover:border-slate-300 dark:hover:border-slate-600 transition-all"
  >
    <svg className="w-4 h-4" fill="none" viewBox="0 0 24 24" stroke="currentColor">
      <path
        strokeLinecap="round"
        strokeLinejoin="round"
        strokeWidth={2}
        d="M4 16v1a3 3 0 003 3h10a3 3 0 003-3v-1m-4-8l-4-4m0 0L8 8m4-4v12"
      />
    </svg>
    重新上传
  </Button>
)}
```

**关键特性**：

- `variant="outline"`：轮廓样式，不抢主要操作的风头
- `size="sm"`：小尺寸，与其他按钮一致
- `gap-2`：图标和文字间距
- `transition-all`：平滑过渡动画
- 条件渲染：只在提供 `onReupload` 时显示

### 3. 回调处理

```typescript
// 在 upload-audio.tsx 中
<TranscriptionResult
  fileName={selectedFile.name}
  language="中文"
  targetLanguage="English"
  segments={segments}
  audioUrl={audioUrl || undefined}
  isTranslating={isTranslating}
  onReupload={handleRemoveFile} // 使用现有的清除函数
/>
```

**`handleRemoveFile` 函数**：

```typescript
const handleRemoveFile = () => {
  setSelectedFile(null);
  setTranscriptionResult(null);
  setTranslationResults(null);
  setIsTranslating(false);
  setShowResult(false);
  setAudioDuration(undefined);
  if (audioUrl) {
    URL.revokeObjectURL(audioUrl);
    setAudioUrl(null);
  }
  if (fileInputRef.current) {
    fileInputRef.current.value = '';
  }
};
```

- ✅ 清除选中的文件

## 🎯 用户体验优化

### 1. 位置优化

**为什么放在最左侧？**

- 重新上传是"返回"操作
- 符合用户从左到右的阅读习惯
- 与"后退"按钮的位置一致

**按钮顺序**：

```
[重新上传] [复制译文] [导出] [发送到对话]
   ↑          ↑        ↑         ↑
 返回操作   次要操作  次要操作  主要操作
```

### 2. 样式优化

**为什么使用 Outline 样式？**

- 区别于主要操作（"发送到对话"使用实心按钮）
- 视觉上不抢眼，但清晰可见
- 符合"次要操作"的设计规范

**颜色选择**：

- 灰色系：中性，不传达强烈情感
- 不使用红色：避免暗示"危险"或"删除"
- 不使用蓝色：避免与主要操作混淆

### 3. 交互优化

**图标选择**：

- 上传箭头（⬆）：直观表达"上传"
- 与初始上传界面的图标一致
- 用户容易理解

**文字标签**：

- "重新上传"：清晰明确
- 不使用"返回"：避免歧义
- 不使用"上传新文件"：太长

### 4. 响应式设计

**移动端优化**：

```typescript
// 可以添加响应式类
className = 'gap-2 ... sm:inline-flex hidden';
// 在小屏幕上可以只显示图标
```

**小屏幕布局**：

```

│ 🎵 文件名.mp3       │
│ 中文 → English      │
│                     │
│ [⬆] [📋] [⬇] [⚡]  │ ← 只显示图标
└─────────────────────┘
```

## 🎨 设计规范总结

### 按钮层级

```
主要操作：实心按钮 + 渐变背景
  例：发送到对话

次要操作：Ghost 按钮 / Outline 按钮
  例：复制译文、导出、重新上传

危险操作：红色按钮
  例：删除（如果有）
```

### 颜色使用

```
蓝色系：主要操作、链接
灰色系：次要操作、中性操作
绿色系：成功状态
红色系：危险操作、错误状态
紫色系：进行中状态
```

### 间距规范

```
按钮间距：gap-2 (0.5rem)
图标文字间距：gap-2 (0.5rem)
按钮内边距：px-4 py-2 (size="sm")
```
