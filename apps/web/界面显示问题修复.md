# 界面显示问题修复

## 🐛 问题描述

在实时录音界面时，本地上传音频的界面也会显示，导致两个界面重叠。

## 🔍 问题原因

使用 `cn()` 函数和 Tailwind 的 `hidden` class 时，可能存在 CSS 优先级问题，导致 `display` 属性没有正确应用。

### 原始代码

```typescript
<div className={cn(mode === 'upload' ? 'block' : 'hidden', 'flex-1 flex flex-col h-full')}>
  <UploadAudio />
</div>

<div className={cn(mode === 'realtime' ? 'block' : 'hidden', 'flex-1 overflow-y-auto ...')}>
  {/* 实时录音内容 */}
</div>
```

**问题**：

- `hidden` class 可能被其他 CSS 规则覆盖
- `flex-1` 等 class 可能影响 `display` 属性
- CSS 优先级导致两个界面同时显示

## ✅ 修复方案

### 使用内联样式强制 display 属性

```typescript
{/* 上传音频界面 */}
<div
  className={mode === 'upload' ? 'flex-1 flex flex-col h-full' : 'hidden'}
  style={{ display: mode === 'upload' ? 'flex' : 'none' }}
>
  <UploadAudio
    onFileSelect={(file) => console.log('Selected file:', file)}
    onStateChange={handleUploadStateChange}
  />
</div>

{/* 实时录音界面 */}
<div
  className={mode === 'realtime' ? 'flex-1 overflow-y-auto p-4 sm:p-6 lg:p-8 custom-scrollbar pb-32' : 'hidden'}
  style={{ display: mode === 'realtime' ? 'block' : 'none' }}
>
  {/* 实时录音内容 */}
</div>
```

### 修复要点

1. **移除 `cn()` 函数**
   - 直接使用三元运算符
   - 避免 class 合并导致的优先级问题

2. **添加内联样式**
   - `style={{ display: ... }}`
   - 内联样式优先级最高，确保正确应用

3. **条件 className**
   - 上传模式：`'flex-1 flex flex-col h-full'`
   - 实时录音模式：`'flex-1 overflow-y-auto ...'`
   - 隐藏时：`'hidden'`

4. **条件 display**
   - 上传模式：`display: 'flex'`（保持 flex 布局）
   - 实时录音模式：`display: 'block'`
   - 隐藏时：`display: 'none'`（强制隐藏）

## 🎨 工作原理

### CSS 优先级

```
内联样式 > ID 选择器 > Class 选择器 > 标签选择器
```

**内联样式优先级最高**，确保 `display: none` 一定生效。

### 显示逻辑

```typescript
// 上传音频模式
mode === 'upload'
  ↓
className: 'flex-1 flex flex-col h-full'
style: { display: 'flex' }
  ↓
显示上传界面，使用 flex 布局

// 实时录音模式
mode === 'realtime'
  ↓
className: 'flex-1 overflow-y-auto ...'
style: { display: 'block' }
  ↓
显示实时录音界面，使用 block 布局

// 隐藏状态
mode !== 当前模式
  ↓
className: 'hidden'
style: { display: 'none' }
  ↓
完全隐藏，不占用空间
```

## 💡 为什么使用内联样式？

### 优势

1. **优先级最高**
   - 确保 `display` 属性一定生效
   - 不会被其他 CSS 规则覆盖

2. **明确性**
   - 代码意图清晰
   - 易于调试

3. **可靠性**
   - 不依赖 CSS class 的加载顺序
   - 不受 Tailwind 配置影响

### 劣势

1. **可维护性**
   - 样式分散在 JSX 和 CSS 中
   - 但对于关键的显示/隐藏逻辑，这是可接受的

2. **性能**
   - 内联样式略微影响性能
   - 但对于只有两个元素的情况，影响可忽略
