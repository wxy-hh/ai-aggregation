# 中英文对照修复说明

## 🐛 问题分析

### 问题 1：中英文翻译错乱

**原因**：

- SiliconFlow API 返回的是**完整的文本**（不是按切片返回）
- 之前的实现：将整段文本一次性翻译，然后按标点分割
- 导致：中文句子和英文句子无法一一对应

**示例**：

```
原文（按句号分割）：
1. 你好，欢迎使用我们的产品。
2. 这是第一个功能介绍。
3. 接下来是第二个功能。

之前的翻译方式（整段翻译）：
"Hello, welcome to our product. This is the first feature introduction. Next is the second feature."

按句号分割后：
1. "Hello, welcome to our product"
2. "This is the first feature introduction"
3. "Next is the second feature"

✅ 看起来对应

但如果翻译结果是：
"Hello and welcome to our product! Here's an introduction to the first feature. The second feature is described next."

按句号分割后：
1. "Hello and welcome to our product!"
2. "Here's an introduction to the first feature"
3. "The second feature is described next"

❌ 对应错乱！
```

### 问题 2：中英文 div 高度不一致

**原因**：

- 之前使用两个独立的列（`grid-cols-2`）
- 左栏和右栏各自独立渲染
- 中文和英文长度不同，导致高度不一致

**示例**：

```
之前的布局：
┌─────────────┬─────────────┐
│ 原文列      │ 译文列      │
├─────────────┼─────────────┤
│ 句子1 (短)  │ 句子1 (长)  │
│             │ 换行了      │
├─────────────┼─────────────┤  ← 不在同一水平线
│ 句子2 (长)  │ 句子2 (短)  │
│ 换行了      │             │
└─────────────┴─────────────┘
```

## ✅ 修复方案

### 修复 1：逐句翻译，确保一一对应

**新的翻译流程**：

```typescript
// 1. 按标点分割成句子
const sentences = transcriptionText.split(/[。！？]/).filter((text) => text.trim().length > 0);

// 2. 逐句翻译
const translatedSentences: string[] = [];
for (let i = 0; i < sentences.length; i++) {
  const sentence = sentences[i].trim();

  const result = await translateText({
    text: sentence,
    sourceLanguage: 'Chinese',
    targetLanguage: 'English',
  });

  translatedSentences.push(result.translatedText.trim());
}

// 3. 合并翻译结果
const fullTranslation = translatedSentences.join('. ') + '.';
```

**优点**：

- ✅ 每个中文句子对应一个英文句子
- ✅ 保证一一对应关系
- ✅ 点击原文列表，对应的英文准确

**示例**：

```
原文句子 1: "你好，欢迎使用我们的产品。"
翻译结果 1: "Hello, welcome to our product."

原文句子 2: "这是第一个功能介绍。"
翻译结果 2: "This is the first feature introduction."

原文句子 3: "接下来是第二个功能。"
翻译结果 3: "Next is the second feature."

✅ 完美对应！
```

### 修复 2：使用 Grid 自动对齐高度

**新的布局方式**：

```typescript
// 每一行包含原文和译文，使用 grid 自动对齐
{segments.map((segment) => (
  <div key={segment.id} className="grid grid-cols-2 gap-6 items-start">
    {/* 左栏 - 原文 */}
    <SegmentBlock segment={segment} showTranslation={false} />

    {/* 右栏 - 译文 */}
    <SegmentBlock segment={segment} showTranslation={true} />
  </div>
))}
```

**关键改进**：

1. **每行一个 grid**：
   - 之前：两个独立的列，各自渲染所有句子
   - 现在：每个句子一行，包含原文和译文

2. **自动高度对齐**：
   - `grid-cols-2`：两列布局
   - `items-start`：顶部对齐
   - Grid 会自动让同一行的元素高度一致

3. **SegmentBlock 填充高度**：
   - 添加 `h-full` 类
   - 确保块填充整个网格单元格

**效果**：

```
新的布局：
┌─────────────┬─────────────┐
│ 原文列      │ 译文列      │
├─────────────┴─────────────┤
│ ┌─────────┐ ┌───────────┐ │
│ │句子1(短)│ │句子1(长)  │ │  ← 同一行，高度一致
│ │         │ │换行了     │ │
│ └─────────┘ └───────────┘ │
├─────────────────────────────┤
│ ┌─────────┐ ┌───────────┐ │

└─────────────────────────────┘
```

## ⚠️ 注意事项

### 翻译速度

**问题**：逐句翻译比整段翻译慢

**原因**：

- 之前：1次 API 调用
- 现在：N次 API 调用（N = 句子数量）

**示例**：

- 10句话：需要10次 API 调用
- 每次调用约1-2秒
- 总时间：10-20秒

**优化方案**（未来）：

1. 并行翻译（同时发送多个请求）
2. 批量翻译（一次调用翻译多句，但要求API返回数组）
3. 缓存翻译结果

### API 调用限制

**问题**：可能触发 API 频率限制

**解决方案**：

- 添加延迟（每次调用间隔100ms）
- 添加重试机制
- 显示翻译进度

### 翻译质量

**优点**：

- ✅ 每句独立翻译，更准确
- ✅ 不会因为上下文导致错乱

**缺点**：

- ⚠️ 可能失去上下文连贯性
- ⚠️ 某些需要上下文的翻译可能不够流畅

**示例**：

```
原文：
1. 他说他会来。
2. 但是他没来。

逐句翻译：
1. "He said he would come."
2. "But he didn't come."

整段翻译（可能更流畅）：
"He said he would come, but he didn't."
```

## 📞 问题排查

### 如果对应关系仍然错乱

**检查点**：

1. **查看控制台日志**：

   ```
   共 10 句需要翻译
   翻译第 1/10 句...
   ```

   确认句子数量是否正确

2. **检查分割逻辑**：

   ```typescript
   const sentences = transcriptionText.split(/[。！？]/);
   ```

   确认标点符号是否正确

3. **检查合并逻辑**：
   ```typescript
   const fullTranslation = translatedSentences.join('. ') + '.';
   ```
   确认分隔符是否正确

### 如果高度仍然不对齐

**检查点**：

1. **检查 Grid 布局**：

   ```typescript
   <div className="grid grid-cols-2 gap-6 items-start">
   ```

   确认每行只有一个 grid

2. **检查 SegmentBlock**：

   ```typescript
   className = '... h-full';
   ```

   确认有 `h-full` 类

3. **检查浏览器开发者工具**：
   - 检查元素的实际高度
   - 查看 Grid 布局是否生效
