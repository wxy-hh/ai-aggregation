# 调试转录结果

## 如何验证是否是 API 识别不全

### 方法1：查看浏览器控制台

1. 打开浏览器开发者工具（F12）
2. 切换到 Console 标签
3. 上传音频文件
4. 查找以下日志：

```
← API 响应成功, 转录文本长度: XXX
```

5. 在控制台输入以下命令查看完整文本：

```javascript
// 查看最近的转录结果
console.log(localStorage.getItem('lastTranscription'));
```

### 方法2：查看 Network 请求

1. 打开浏览器开发者工具（F12）
2. 切换到 Network 标签
3. 上传音频文件
4. 找到 `transcribe` 请求
5. 点击查看 Response
6. 查看返回的 `transcription` 字段

**示例**：

```json
{
  "id": "temp-id",
  "status": "completed",
  "transcription": "这里是完整的转录文本...",
  "message": "转录成功"
}
```

### 方法4：对比音频内容

1. 播放音频文件
2. 手动记录音频内容
3. 对比转录文本
4. 找出缺失的部分

**记录格式**：


## 可能导致识别不全的原因

### 1. 音频质量问题 ⭐⭐⭐⭐⭐

**常见问题**：

- 背景噪音太大
- 音量过小或过大
- 说话不清晰
- 语速过快
- 口音太重

**解决方案**：

- 使用高质量录音设备
- 在安静环境录音
- 说话清晰、语速适中
- 音量适中

### 2. 音频格式问题 ⭐⭐⭐

**可能问题**：

- 采样率太低
- 比特率太低
- 编码格式不佳

**建议格式**：

- 格式：MP3 或 WAV
- 采样率：≥16kHz
- 比特率：≥128kbps

**检查方法**：

```bash
# 使用 ffprobe 查看音频信息
ffprobe 音频文件.mp3

# 或使用在线工具
# https://www.metadata2go.com/
```

### 3. 模型限制 ⭐⭐⭐⭐

**SenseVoiceSmall 的限制**：

- 这是一个"Small"模型（较小的模型）
- 识别准确率可能不如大模型
- 对某些场景处理不佳

**对比其他模型**：

```
模型大小    | 准确率  | 速度
-----------|---------|-------
Small      | 85-90%  | 快
Medium     | 90-95%  | 中
Large      | 95-98%  | 慢
```

### 4. 音频时长问题 ⭐⭐

**可能问题**：

- 音频太长（>1小时）
- API 超时
- 内存不足

**你的情况**：

- 音频：2分22秒 ✅ 正常范围
- 不太可能是时长问题

### 5. API 参数问题 ⭐

**当前参数**：

```typescript
{
  model: 'FunAudioLLM/SenseVoiceSmall',
  file: audio_file
}
```

**可能的改进**：

- 查看 API 文档是否有其他参数
- 例如：language, prompt, temperature 等
