3. 等待处理完成
4. 确认结果界面显示

**预期结果**：

- ✅ 转录结果显示

### 步骤 2：重新上传

1. **打开浏览器控制台**（F12 或 Cmd+Option+I）
2. 点击"重新上传"按钮
3. **观察**：
   - ✅ 文件选择器弹出
   - ✅ **没有返回上传界面**（重要！）
4. 选择同一个或不同的音频文件
5. **观察**：
   - ✅ **停留在结果界面**（重要！）
   - ✅ 显示 Loading 遮罩
   - ✅ 遮罩覆盖在结果界面上
   - ✅ 音频图标有动画
   - ✅ 进度条左右滑动
6. 等待处理完成
7. **观察**：
   - ✅ Loading 遮罩消失
   - ✅ 结果直接更新
   - ✅ **无页面跳转**（重要！）

### 步骤 3：多次重新上传

1. 重复步骤 2 两到三次
2. 每次都确认：
   - ✅ 停留在结果界面

## ❌ 失败情况

### 如果返回上传界面

**现象**：

- 点击"重新上传"后，看到上传界面（拖拽区域）
- 结果界面消失

**原因**：

- `showResult` 被设置为 `false`

**检查**：

```javascript
// 在控制台搜索
'setShowResult(false)';
```

**解决**：

- 确认代码中已移除 `setShowResult(false)`

### 如果 Loading 不显示

**现象**：

- 选择文件后，没有 Loading 遮罩
- 界面空白或显示旧内容

**原因**：

- `isProcessing` 状态不正确

**检查**：

```javascript
// 在控制台输入
console.log('isProcessing:', uploadMutation.isPending);
```

**解决**：

- 确认 `isProcessingNewFile` 逻辑正确

### 如果结果不更新

**现象**：

- 处理完成后，仍然显示旧内容

**原因**：

- 状态没有正确更新

**检查**：

- 查看控制台是否有错误
- 确认转录和翻译 API 调用成功
