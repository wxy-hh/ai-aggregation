# 切换标签页保持上传状态

## 🎯 用户需求

> "现在从上传音频的界面切换到实时录音界面，由于上传的音频还没有处理完成，我希望可以切换，但是再次切换回来的时候要展示处理的结果，不要切换回来还是原始的上传音频界面"

## ✨ 实现方案

### 核心问题

**之前的实现**：

```typescript
{mode === 'upload' ? (
  <UploadAudio />  // ❌ 切换时组件被卸载，状态丢失
) : (
  <RealtimeRecording />
)}
```

**问题**：

- ❌ 切换到实时录音时，`UploadAudio` 组件被卸载
- ❌ 所有状态（文件、转录结果、翻译结果）丢失
- ❌ 切换回来时回到初始的上传界面
- ❌ 正在处理的任务被中断

### 解决方案

**使用 CSS 隐藏而不是条件渲染**：

```typescript
{/* 上传音频界面 - 使用 CSS 隐藏 */}
<div className={cn(mode === 'upload' ? 'block' : 'hidden')}>
  <UploadAudio />  // ✅ 组件保持挂载，状态保留
</div>

{/* 实时录音界面 - 使用 CSS 隐藏 */}
<div className={cn(mode === 'realtime' ? 'block' : 'hidden')}>
  <RealtimeRecording />
</div>
```

**优势**：

- ✅ 组件始终保持挂载
- ✅ 状态不会丢失
- ✅ 正在处理的任务继续执行
- ✅ 切换回来时显示最新状态

## 🔍 技术细节

### 为什么使用 CSS 隐藏？

**CSS 隐藏**：

```typescript
<div className={cn(mode === 'upload' ? 'block' : 'hidden')}>
  <Component />
</div>
```

**优势**：

- ✅ 组件保持挂载
- ✅ 状态保留
- ✅ 事件监听器保留
- ✅ 异步任务继续执行

**条件渲染**：

```typescript
{mode === 'upload' && <Component />}
```

**问题**：

- ❌ 组件被卸载
- ❌ 状态丢失
- ❌ 事件监听器移除
- ❌ 异步任务中断

### 状态同步机制

```typescript
// 子组件（UploadAudio）
useEffect(() => {
  if (onStateChange) {
    onStateChange({
      hasUploadedFile: selectedFile !== null,
      isProcessing: uploadMutation.isPending || isTranslating,
      showResult,
    });
  }
}, [selectedFile, uploadMutation.isPending, isTranslating, showResult]);

// 父组件（VoicePage）
<UploadAudio
  onStateChange={(state) => {
    setUploadState(state); // 更新父组件状态
  }}
/>
```

**流程**：

1. 子组件状态变化
2. 触发 `useEffect`
3. 调用 `onStateChange` 回调
4. 父组件更新 `uploadState`
5. 标签页指示器更新

### 性能考虑

**问题**：两个界面都保持挂载，会不会影响性能？

**答案**：影响很小

**原因**：

1. 隐藏的组件不会触发重新渲染（除非状态变化）
2. 隐藏的组件不会执行布局计算
3. 隐藏的组件不会绘制到屏幕
4. 只有 DOM 节点保留在内存中

**优化**：

- 使用 `display: none`（`hidden` class）而不是 `visibility: hidden`
- `display: none` 完全移除元素的布局计算
