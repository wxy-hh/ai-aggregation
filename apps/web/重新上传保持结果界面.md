# 重新上传保持结果界面

## 🎯 需求

用户要求：

> "现在重新上传的时候，我希望直接覆盖上一个展示的结果，不要再回到最初上传的界面了，在上传处理的过程中，页面可以一直停留当前结果界面，中间加一个loading"

## ✨ 实现方案

### 核心改进

**之前的行为**：

```
点击"重新上传"
  ↓
选择文件
  ↓
setShowResult(false) ❌ 返回上传界面
  ↓
开始处理
  ↓
处理完成
  ↓
setShowResult(true) → 显示结果界面
```

**现在的行为**：

```
点击"重新上传"
  ↓
选择文件
  ↓
保持 showResult = true ✅ 停留在结果界面
  ↓
显示 Loading 遮罩 ✅ 覆盖在结果界面上
  ↓
开始处理（转录 + 翻译）
  ↓
处理完成
  ↓
隐藏 Loading 遮罩 ✅ 直接显示新结果
```

## 💡 技术细节

### 为什么不设置 `showResult = false`？

**原因**：

1. `showResult` 控制是否显示结果界面
2. 设置为 `false` 会导致返回上传界面
3. 我们希望停留在结果界面，所以保持 `true`

### 如何判断是否正在处理？

```typescript
const isProcessingNewFile = uploadMutation.isPending || (!transcriptionResult && showResult);
```

**逻辑**：

- `uploadMutation.isPending`：正在上传/转录
- `!transcriptionResult && showResult`：在结果界面但没有转录结果（说明正在处理新文件）

### 为什么需要占位文本？

```typescript
const displayText = transcriptionResult || '正在处理音频，请稍候...';
```

**原因**：

1. 重新上传时，`transcriptionResult` 被清空（设为 `null`）
2. 但我们仍然在结果界面，需要显示一些内容
3. 占位文本提供友好的提示
