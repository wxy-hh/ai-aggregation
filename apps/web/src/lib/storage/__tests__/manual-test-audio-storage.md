# 音频存储功能手动测试指南

## 测试目标

验证音频历史记录的智能存储策略：

- ✅ 最近10条保留音频
- ✅ 超过10条只保留文本
- ✅ 历史记录切换正常工作

## 测试步骤

### 1. 准备测试音频文件

准备至少 12 个测试音频文件（可以是同一个文件的副本）：

- test-audio-1.mp3
- test-audio-2.mp3
- ...
- test-audio-12.mp3

### 2. 上传音频文件

1. 打开应用：`http://localhost:3000/voice`
2. 切换到"上传音频"标签
3. 依次上传 12 个音频文件
4. 等待每个文件转录和翻译完成

### 3. 验证最近10条有音频

1. 打开右侧历史记录面板
2. 点击最近的 10 条记录
3. **预期结果**：
   - ✅ 音频播放器正常显示
   - ✅ 可以播放音频
   - ✅ 转录和翻译文本正常显示

### 4. 验证超过10条没有音频

1. 点击第 11 条和第 12 条历史记录
2. **预期结果**：
   - ✅ 显示"音频文件不可用"提示
   - ✅ 转录和翻译文本仍然正常显示
   - ✅ 显示"重新上传"按钮

### 5. 验证新上传触发清理

1. 上传第 13 个音频文件
2. 等待处理完成
3. 检查历史记录：
   - ✅ 第 13 条（最新）有音频
   - ✅ 第 4-13 条有音频（共10条）
   - ✅ 第 1-3 条没有音频

### 6. 验证浏览器开发者工具

打开浏览器开发者工具（F12）：

1. 切换到 Application/Storage 标签
2. 找到 IndexedDB → AudioHistoryDB → audioHistory
3. 查看记录：
   - ✅ 最近10条有 `audioBlob` 字段
   - ✅ 超过10条的 `audioBlob` 字段为空或未定义

## 控制台日志检查

在上传和切换历史记录时，应该看到以下日志：

```
[IndexedDBStorage] Cleaned up X old audio blobs
[UploadAudio] Audio restored from blob
[UploadAudio] Audio not available for this history item
```

## 常见问题排查

### 问题1：所有历史记录都没有音频

**可能原因**：

- 浏览器不支持 Blob 存储
- IndexedDB 配额已满

**解决方法**：

1. 检查浏览器控制台是否有错误
2. 清除浏览器数据后重试
3. 使用现代浏览器（Chrome、Firefox、Edge）

### 问题2：清理没有触发

**可能原因**：

- 清理逻辑有 bug
- 数据库版本不匹配

**解决方法**：

1. 检查控制台日志
2. 清除 IndexedDB 数据库
3. 刷新页面重新测试

### 问题3：历史记录切换后音频不播放

**可能原因**：

- Blob URL 没有正确创建
- 音频格式不支持

**解决方法**：

1. 检查 `audioUrl` 状态
2. 查看控制台是否有错误
3. 确认音频文件格式正确

## 性能测试

### 存储空间检查

在浏览器控制台运行：

```javascript
// 检查存储使用情况
navigator.storage.estimate().then((estimate) => {
  console.log('已使用:', (estimate.usage / 1024 / 1024).toFixed(2), 'MB');
  console.log('总配额:', (estimate.quota / 1024 / 1024).toFixed(2), 'MB');
  console.log('可用:', ((estimate.quota - estimate.usage) / 1024 / 1024).toFixed(2), 'MB');
});
```

### 预期结果

- 10个音频文件（每个约5MB）：约 50MB
- 文本数据：< 1MB
- 总计：约 50-51MB
