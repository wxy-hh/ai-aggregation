# 语法错误修复

## 🐛 错误描述

在 `apps/web/src/app/voice/page.tsx` 第 188 行出现错误：

```
src/app/voice/page.tsx (188:17) @ onStateChange
> 188 |                 setUploadState(state);
```

## 🔍 问题原因

代码中有一个多余的 `});` 导致语法错误：

```typescript
const handleUploadStateChange = useCallback((state) => {
  console.log('📊 父组件收到状态更新:', state);
  setUploadState(state);
}, []);
});  // ❌ 多余的闭合括号
```

## ✅ 修复方案

### 1. 添加 useCallback import

```typescript
import { useState, useCallback } from 'react';
```

### 2. 创建优化的回调函数

```typescript
const handleUploadStateChange = useCallback(
  (state: { hasUploadedFile: boolean; isProcessing: boolean; showResult: boolean }) => {
    console.log('📊 父组件收到状态更新:', state);
    setUploadState(state);
  },
  []
);
```

### 3. 移除多余的括号

```typescript
// 修复前
}, []);
});  // ❌ 删除这行

// 修复后
}, []);  // ✅ 正确
```

### 4. 使用回调函数

```typescript
<UploadAudio
  onFileSelect={(file) => console.log('Selected file:', file)}
  onStateChange={handleUploadStateChange}  // ✅ 使用 useCallback 优化的函数
/>
```

## 🎯 优化效果

### 使用 useCallback 的优势

1. **避免不必要的重新渲染**
   - 回调函数引用保持稳定
   - 子组件不会因为父组件重新渲染而重新渲染

2. **性能优化**
   - 减少函数创建次数
   - 减少内存占用

3. **依赖优化**
   - 明确声明依赖项（空数组 `[]`）
   - 函数只创建一次
