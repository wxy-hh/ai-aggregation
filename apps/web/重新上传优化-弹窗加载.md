# 重新上传优化 - 弹窗加载

## 🎯 优化说明

将"重新上传"功能从"返回上传界面"改为"直接弹出文件选择器 + Loading 遮罩"，提供更流畅的用户体验。

## ✨ 优化前 vs 优化后

### 优化前的流程

```
点击"重新上传"
  ↓
清除所有数据
  ↓
返回上传界面 ❌ 页面跳转，体验不连贯
  ↓
用户再次点击"选择文件"
  ↓
选择文件
  ↓
开始处理
```

**问题**：

- ❌ 需要两次点击（重新上传 + 选择文件）
- ❌ 页面跳转，视觉不连贯
- ❌ 用户需要重新定位上传区域

### 优化后的流程

```
点击"重新上传"
  ↓
直接弹出文件选择器 ✅ 一步到位
  ↓
用户选择文件
  ↓
显示 Loading 遮罩 ✅ 保持在当前页面
  ↓
处理完成
  ↓
更新结果 ✅ 无缝切换
```

**优势**：

- ✅ 只需一次点击
- ✅ 无页面跳转，体验流畅
- ✅ Loading 状态清晰可见
- ✅ 保持用户在当前上下文

## 🔧 实现细节

### 1. 修改重新上传逻辑

```typescript
// 之前：清除数据并返回上传界面
const handleRemoveFile = () => {
  setSelectedFile(null);
  setShowResult(false);
  // ... 清除所有数据
};

// 现在：直接触发文件选择器
const handleReupload = () => {
  fileInputRef.current?.click(); // 直接打开文件选择器
};
```

**优势**：

- 一步到位，无需返回上传界面
- 用户体验更流畅

### 2. 添加 Loading 状态

```typescript
interface TranscriptionResultProps {
  // ... 其他 props
  isProcessing?: boolean; // 新增：是否正在处理
  onReupload?: () => void;
}

// 传递处理状态
<TranscriptionResult
  isProcessing={uploadMutation.isPending} // 使用 mutation 的 pending 状态
  onReupload={handleReupload}
/>
```

## 💡 扩展功能

### 功能1：显示处理进度

```typescript
const [progress, setProgress] = useState(0);

// 模拟进度更新
useEffect(() => {
  if (isProcessing) {
    const interval = setInterval(() => {
      setProgress(prev => Math.min(prev + 10, 90));
    }, 1000);
    return () => clearInterval(interval);
  }
}, [isProcessing]);

// 显示进度
<div className="w-full bg-slate-100 rounded-full h-2">
  <div
    className="h-full bg-gradient-to-r from-blue-500 to-indigo-600 transition-all"
    style={{ width: `${progress}%` }}
  />
</div>
```

### 功能2：显示处理步骤

```typescript
const steps = [
  { label: '上传文件', status: 'completed' },
  { label: '音频转录', status: 'processing' },
  { label: '文本翻译', status: 'pending' },
];

// 显示步骤列表
{steps.map(step => (
  <div key={step.label} className="flex items-center gap-2">
    {step.status === 'completed' && <CheckIcon />}
    {step.status === 'processing' && <SpinnerIcon />}
    {step.status === 'pending' && <ClockIcon />}
    <span>{step.label}</span>
  </div>
))}
```

const [abortController, setAbortController] = useState<AbortController | null>(null);
