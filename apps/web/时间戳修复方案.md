# 时间戳修复方案

## 问题总结

1. **音频识别不全** - 转录文本比实际音频内容少
2. **时间戳不准确** - 使用的是模拟时间（每句5秒）
3. **时长不匹配** - 音频2分22秒，文本只到50秒

## 根本原因

**SiliconFlow API 不支持时间戳**

- API 只返回纯文本，没有时间信息
- 当前使用固定5秒/句的模拟时间戳
- 无法获取真实的说话时间

## 立即修复方案

### 修改 `createSegments` 函数

在 `apps/web/src/components/voice/upload-audio.tsx` 中：

```typescript
/**
 * 创建转录片段数据
 * 根据音频总时长和文本长度智能分配时间戳
 */
const createSegments = (
  transcriptionText: string,
  translationText?: string,
  audioDuration?: number // 新增：音频总时长（秒）
) => {
  const sentences = transcriptionText.split(/[。！？]/).filter((text) => text.trim().length > 0);

  let translationSentences: string[] = [];
  if (translationText) {
    translationSentences = translationText.split(/[.!?]/).filter((text) => text.trim().length > 0);
  }

  if (sentences.length === 0) {
    return [
      {
        id: '1',
        timestamp: '00:00:00',
        speaker: 'Speaker A',
        speakerLabel: 'Speaker A' as const,
        originalText: transcriptionText,
        translatedText: translationText || 'Translation in progress...',
        startTime: 0,
        endTime: audioDuration || 10,
      },
    ];
  }

  // 检查句子数量匹配
  if (translationSentences.length !== sentences.length && translationText) {
    console.warn('⚠ 中英文句子数量不匹配:');
    console.warn(`  中文: ${sentences.length} 句`);
    console.warn(`  英文: ${translationSentences.length} 句`);
  }

  // 🔧 核心修复：根据文本长度智能分配时间
  const charCounts = sentences.map((s) => s.length);
  const totalChars = charCounts.reduce((sum, count) => sum + count, 0);

  // 如果没有音频时长，估算（平均每个字0.5秒）
  const estimatedDuration = audioDuration || totalChars * 0.5;

  let currentTime = 0;

  return sentences.map((sentence, index) => {
    // 根据字符数比例分配时间（长句子分配更多时间）
    const charRatio = charCounts[index] / totalChars;
    const duration = estimatedDuration * charRatio;

    const startTime = currentTime;
    const endTime = currentTime + duration;
    currentTime = endTime;

    // 格式化时间戳
    const minutes = Math.floor(startTime / 60);
    const seconds = Math.floor(startTime % 60);

    // 智能匹配翻译
    let translatedText: string;
    if (!translationText) {
      translatedText = 'Translation in progress...';
    } else if (translationSentences[index]) {
      translatedText = translationSentences[index].trim() + '.';
    } else if (translationSentences.length > 0) {
      const remainingTranslations = translationSentences.slice(index);
      translatedText =
        remainingTranslations.length > 0
          ? remainingTranslations.join('. ') + '.'
          : '(Translation alignment mismatch)';
    } else {
      translatedText = 'Translation in progress...';
    }

    return {
      id: String(index + 1),
      timestamp: `${String(minutes).padStart(2, '0')}:${String(seconds).padStart(2, '0')}:00`,
      speaker: index % 3 === 0 ? 'Speaker A' : index % 3 === 1 ? 'Speaker B' : 'Speaker C',
      speakerLabel: (index % 3 === 0
        ? 'Speaker A'
        : index % 3 === 1
          ? 'Speaker B'
          : 'Speaker C') as const,
      originalText: sentence.trim() + '。',
      translatedText,
      startTime,
      endTime,
    };
  });
};
```

## 获取真实音频时长

### 方案1：在 TranscriptionResult 组件中获取

修改 `apps/web/src/components/voice/transcription-result.tsx`：

```typescript
export function TranscriptionResult({
  fileName,
  language,
  targetLanguage,
  segments,
  audioUrl,
  isTranslating = false,
  onAudioDurationLoaded, // 新增回调
}: TranscriptionResultProps) {
  const audioRef = useRef<HTMLAudioElement>(null);

  useEffect(() => {
    const audio = audioRef.current;
    if (!audio) return;

    const handleLoadedMetadata = () => {
      const duration = audio.duration;
      console.log('✓ 音频时长:', duration, '秒');

      // 通知父组件音频时长
      if (onAudioDurationLoaded) {
        onAudioDurationLoaded(duration);
      }
    };

    audio.addEventListener('loadedmetadata', handleLoadedMetadata);

    return () => {
      audio.removeEventListener('loadedmetadata', handleLoadedMetadata);
    };
  }, [onAudioDurationLoaded]);

  // ... 其余代码
}
```

## 关于问题1：识别不全

### 检查方法

1. 在浏览器控制台查看完整转录文本：

   ```
   console.log('转录文本:', result.transcription);
   ```

2. 手动对比音频内容，确认是否真的丢失

### 可能原因

1. **音频质量问题**
   - 背景噪音太大
   - 音量过小
   - 说话不清晰

2. **模型限制**
   - SenseVoiceSmall 对某些音频处理不佳
   - 可能需要更强大的模型

3. **音频格式问题**
   - 某些格式转录效果不好
   - 建议使用 MP3 或 WAV

### 解决方案

如果确实存在识别不全的问题：

1. **短期方案**：
   - 提高音频质量
   - 尝试重新上传
   - 手动补充缺失内容

2. **长期方案**：
   - 切换到更强大的模型（如 Whisper）
   - 使用讯飞语音识别（你已有 API）
   - 添加音频预处理（降噪、增强）
