# 测试界面显示修复

## 🎯 测试目标

验证在切换标签页时，只显示一个界面，不会出现重叠。

## 🚀 快速测试

### 测试 1：实时录音模式

1. 访问：http://localhost:3002/voice
2. 默认在"实时录音"标签页
3. **观察页面**：
   - ✅ 只显示实时录音界面
   - ✅ 看到波形可视化
   - ✅ 看到转录列表
   - ✅ 看到录音控制按钮
   - ❌ 不应该看到上传音频的拖拽区域

### 测试 2：上传音频模式

1. 点击"上传音频"标签
2. **观察页面**：
   - ✅ 只显示上传音频界面
   - ✅ 看到拖拽上传区域
   - ✅ 看到"选择文件"按钮
   - ❌ 不应该看到实时录音的波形和转录列表

### 测试 3：来回切换

1. 点击"实时录音"标签
2. 点击"上传音频"标签
3. 重复 5 次
4. **观察**：
   - ✅ 每次只显示一个界面
   - ✅ 切换流畅，无闪烁
   - ✅ 无重叠现象

### 测试 4：上传后切换

1. 在"上传音频"标签上传文件
2. 等待处理开始（看到 Loading）
3. 切换到"实时录音"标签
4. **观察**：
   - ✅ 只显示实时录音界面
   - ✅ "上传音频"标签显示蓝色脉冲指示器
   - ❌ 不应该看到上传界面的 Loading 遮罩
5. 切换回"上传音频"标签
6. **观察**：
   - ✅ 只显示上传音频界面
   - ✅ 看到处理结果或 Loading 遮罩

## 🔍 使用开发者工具验证

### 步骤 1：打开开发者工具

1. 按 F12 或 Cmd+Option+I
2. 切换到 Elements 标签

### 步骤 2：检查实时录音模式

1. 确保在"实时录音"标签页
2. 在 Elements 中找到两个主要的 div
3. **上传音频界面的 div**：
   ```html
   <div class="hidden" style="display: none;">
     <!-- 应该是折叠的 -->
   </div>
   ```
4. **实时录音界面的 div**：
   ```html
   <div class="flex-1 overflow-y-auto ..." style="display: block;">
     <!-- 应该是展开的 -->
   </div>
   ```

### 步骤 3：检查上传音频模式

1. 切换到"上传音频"标签页
2. **上传音频界面的 div**：
   ```html
   <div class="flex-1 flex flex-col h-full" style="display: flex;">
     <!-- 应该是展开的 -->
   </div>
   ```
3. **实时录音界面的 div**：
   ```html
   <div class="hidden" style="display: none;">
     <!-- 应该是折叠的 -->
   </div>
   ```

## ❌ 失败情况

### 如果看到两个界面重叠

**现象**：

- 同时看到上传界面和实时录音界面
- 界面重叠在一起

**检查**：

1. 打开开发者工具
2. 检查两个 div 的 `display` 属性
3. 如果都不是 `none`，说明修复没有生效

**解决**：

- 确认代码已正确更新
- 清除浏览器缓存
- 重启开发服务器

### 如果切换时有闪烁

**现象**：

- 切换标签页时界面闪烁
- 看到短暂的重叠

**原因**：

- 可能是 React 渲染时序问题
- 或者 CSS 过渡效果

**检查**：

- 这是正常的，因为 React 需要时间更新 DOM
- 如果闪烁明显，可以添加过渡动画
