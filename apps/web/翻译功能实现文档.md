# 翻译功能实现文档

## 概述

使用讯飞Lite模型为语音转录自动提供中英文翻译功能。

## 架构设计

### 工作流程

1. 用户上传音频文件
2. SiliconFlow API 将音频转录为中文文本
3. **新增**: 自动调用讯飞Lite API 将中文翻译为英文
4. 在分栏界面中展示双语结果

### 核心组件

#### 1. 翻译 API 路由

**文件**: `apps/web/src/app/api/voice/translate/route.ts`

- 接口地址: `POST /api/voice/translate`
- 使用讯飞Lite模型 (`lite`)
- 温度参数: 0.3 (保证翻译稳定性)
- 最大令牌数: 4096

**请求格式**:

```json
{
  "text": "要翻译的中文文本",
  "sourceLanguage": "Chinese",
  "targetLanguage": "English"
}
```

**响应格式**:

```json
{
  "translatedText": "翻译后的英文文本",
  "sourceLanguage": "Chinese",
  "targetLanguage": "English",
  "usage": {
    "promptTokens": 123,
    "completionTokens": 456,
    "totalTokens": 579
  }
}
```

#### 2. 翻译客户端库

**文件**: `apps/web/src/lib/api/translation.ts`

- `translateText()` 函数用于客户端 API 调用
- 完整的错误处理和类型安全
- 可在多个组件中复用

#### 3. 上传组件更新

**文件**: `apps/web/src/components/voice/upload-audio.tsx`

**新增状态**:

- `translationResult`: 存储翻译文本
- `isTranslating`: 翻译加载状态

**新增函数**:

- `createSegments()`: 替代 `createMockSegments()`
  - 将转录文本分割为句子
  - 将翻译文本分割为句子
  - 按索引匹配句子
  - 创建双语片段

**工作流程**:

```
上传 → 转录 → 显示界面 → 翻译 → 更新界面
```

#### 4. 结果组件更新

**文件**: `apps/web/src/components/voice/transcription-result.tsx`

**新增属性**:

- `isTranslating?: boolean` - 显示"翻译中..."徽章

**界面更新**:

- 状态徽章显示翻译进度
- 片段实时更新为真实翻译
- 从"Translation in progress..."平滑过渡到实际翻译

## 翻译策略

### 选择的方案: 全文翻译

✅ **一次性翻译整个转录文本**

**优点**:

- 实现简单
- 速度快（单次 API 调用）
- 成本低
- 适合短到中等长度的音频

**缺点**:

- 超长音频可能触及令牌限制
- 单点故障（翻译失败影响全部内容）

### 备选方案（未实现）

1. **逐句翻译**
   - 更强的错误处理能力
   - 可显示渐进式结果
   - 更多 API 调用 = 更高成本

2. **专用翻译 API** (DeepL、Google Translate)
   - 专门针对翻译优化
   - 需要额外的 API 配置

## 错误处理

### 翻译失败

- 非阻塞式: 转录结果仍然显示
- 用户通知: 显示警告提示
- 优雅降级: 显示"Translation in progress..."占位符

### 网络错误

- 捕获并记录错误
- 用户友好的错误消息
- 提供重试选项

## 界面/交互特性

### 加载状态

1. **转录中**: 蓝色徽章"转录中..."
2. **翻译中**: 紫色徽章"翻译中..." (动画脉冲效果)
3. **已完成**: 绿色徽章"已完成"

### 显示模式

- **仅原文**: 仅显示中文
- **仅译文**: 仅显示英文
- **双语对照**: 左右分栏显示（默认）

### 片段匹配

- 按中文标点分割: `。！？`
- 按英文标点分割: `.!?`
- 按索引匹配（第1句 → 第1句）
- 优雅处理数量不匹配的情况

## 性能考虑

### 令牌限制

- 讯飞Lite 最大令牌数: 4096
- 平均每个中文字符 ≈ 2 个令牌
- 安全限制: 每次请求约 2000 个中文字符

### 优化机会

1. **缓存**: 在数据库中存储翻译结果
2. **批处理**: 将短片段分组处理
3. **流式传输**: 实时显示翻译结果
4. **重试逻辑**: 失败时自动重试

## 故障排除

### 翻译未启动

- 检查 `.env.local` 中的 `XUNFEI_API_PASSWORD`
- 验证 API 密钥是否有效
- 检查网络连接
- 查看浏览器控制台日志

### 翻译质量差

- 调整温度参数（尝试 0.1-0.5）
- 检查源文本质量
- 考虑使用不同的模型
- 在提示词中添加上下文

### 令牌限制错误

- 将长文本分块处理
- 实现逐句翻译
- 如果允许，增加 max_tokens
