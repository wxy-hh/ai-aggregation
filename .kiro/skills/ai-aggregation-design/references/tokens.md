## 1. 字体系统

### 字体栈

| 角色 | 字体 | 后备 | 用途 |
| --- | --- | --- | --- |
| Display | `"Space Grotesk"` | `system-ui, sans-serif` | 大标题、区块标题、强调数字 |
| Body | `"DM Sans"` | `system-ui, sans-serif` | 正文、按钮、表单、标签 |
| Mono | `"JetBrains Mono"` | `ui-monospace, monospace` | 技术型占位、少量代码示例 |

### Mono 规则

`mono_for_code: false`  
`mono_for_metrics: false`

这个品牌的数字和状态并不走终端感路线。即使页面里存在代码块或任务 ID，默认也仍应优先使用正文无衬线；等到明确出现开发者上下文时，再局部启用 mono。

## 4. 圆角与描边

| Token | Value | 用途 |
| --- | --- | --- |
| `--radius-element` | 8px | 小部件、色块、图标基座 |
| `--radius-control` | 12px | 按钮、输入、标签底 |
| `--radius-component` | 24px | 普通卡片、模块容器 |
| `--radius-container` | 32px | 弹层、抽屉、外层大面板 |
| `--radius-pill` | 999px | 胶囊标签、分段控件 |

角哲学是“软，但不糯”。小控件不做苹果式极圆，也不做锐角工业风；真正的大圆角只留给工作台级容器。

## 5. 阴影与玻璃

| Level | Light | Dark | 用途 |
| --- | --- | --- | --- |
| `--shadow-1` | `0 1px 2px rgba(15, 23, 42, 0.06)` | `0 2px 10px rgba(0, 0, 0, 0.18)` | 输入、轻卡片 |
| `--shadow-2` | `0 8px 20px rgba(76, 95, 154, 0.10)` | `0 14px 32px rgba(0, 0, 0, 0.28)` | 浮层按钮、次级面板 |
| `--shadow-3` | `0 20px 60px -10px rgba(59, 130, 246, 0.10)` | `0 24px 56px rgba(0, 0, 0, 0.40)` | Hero、重点容器、模态 |

策略属于轻量 Material。深度来自半透明白面 + 细描边 + 柔和阴影，而不是靠黑重阴影砸出来。

## 6. 动效

| 项目 | 值 |
| --- | --- |
| personality | `smooth` |
| easing | `cubic-bezier(0.2, 0.8, 0.2, 1)` |
| fast | `180ms` |
| normal | `240ms` |
| slow | `480ms` |

所有交互都要轻，不要跳。hover 最多做轻微抬升、亮度提升、边框提亮；不要出现夸张弹性。

## 7. Hero Stage

- `medium`: `mesh`
- `color_mode`: `palette`
- `saturation`: `muted`
- `motion`: `pulse`
- `subject`: `none`
- `relation`: `flat`

这套语言里的“英雄区”本质上是一块被柔光包围的大工作台，不是一个独立 3D 物体。背景层只负责制造氛围和聚焦，不抢文字和主操作。
