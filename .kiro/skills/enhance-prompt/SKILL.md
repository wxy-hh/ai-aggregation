# 提示词增强技能

这个技能专注于优化和增强设计提示词，将模糊的想法转换为清晰、具体、适合 Stitch 生成的高质量提示词。

## 主要功能

### 1. 具体性增强

- 将模糊的描述转换为具体的设计要求
- 补充缺失的设计细节
- 明确设计意图和目标

### 2. 关键词注入

- 添加相关的 UI/UX 专业术语
- 注入设计模式和最佳实践
- 使用 Stitch 友好的描述语言

### 3. 上下文整合

- 引入设计系统的上下文信息
- 保持与现有设计的一致性
- 考虑品牌和风格指南

### 4. 结构化输出

- 组织提示词为清晰的结构
- 分离功能需求和视觉需求
- 优化提示词的可读性

## 使用场景

当你需要进行以下工作时，使用此技能：

- 将初步想法转换为可执行的设计提示
- 优化现有提示词以获得更好结果
- 为团队成员提供提示词模板
- 标准化设计请求的表达方式

## 工作流程

1. **输入分析**：理解用户的原始想法和需求
2. **具体化**：将模糊描述转换为具体要求
3. **关键词增强**：添加专业的 UI/UX 术语
4. **上下文注入**：整合设计系统信息
5. **结构化**：组织为清晰的提示词格式
6. **验证优化**：确保提示词完整且可执行

## 前置要求

- 基本的设计需求或想法
- 了解目标应用的类型和风格
- 如有设计系统文档，提供相关信息

## 提示词增强示例

**原始输入**：

```
做一个登录页面
```

**增强后**：

```
创建一个现代化的登录页面，包含以下元素：
- 居中的登录表单卡片，带有轻微阴影
- 邮箱和密码输入框，使用圆角设计
- 主要操作按钮（登录），使用品牌主色
- "忘记密码"和"注册"链接
- 可选的社交登录选项（Google、GitHub）
- 响应式布局，适配移动端和桌面端
- 简洁的背景，可使用渐变或几何图案
- 遵循 Material Design 或现代扁平化风格
```

## 最佳实践

- 提供尽可能多的上下文信息
- 明确设计的目标用户和使用场景
- 参考现有的设计系统或风格指南
- 使用具体的视觉描述而非抽象概念
- 分离功能需求和视觉需求
- 考虑响应式和可访问性要求
